01

Python + Production APIs

Weeks 1–2 · Build Phase

Master the production Python skills that separate hobbyists from professionals. You'll build async APIs, handle webhooks from real services, and write code that's tested, typed, and deployment-ready from day one.

🎯 Learning Objectives

  • Build async REST APIs with FastAPI and Pydantic validation
  • Design webhook endpoints that handle events from Stripe, Twilio, and SendGrid
  • Implement proper error handling, retries, and idempotency
  • Write comprehensive tests with pytest and async fixtures
  • Use type hints and Pydantic models for bulletproof data contracts

📚 Key Topics

  • FastAPI routers, middleware, dependency injection
  • Async/await patterns with httpx
  • REST + GraphQL API design principles
  • Third-party SDK integration (Stripe, Twilio, SendGrid)
  • Environment management, secrets, .env best practices

🛠 Hands-On Project

Build a production webhook server that processes events from 3 services (Stripe payments, Twilio SMS, SendGrid emails) with proper validation, idempotency, error handling, and a test suite achieving 90%+ coverage.

Python 3.12+ FastAPI httpx Pydantic pytest
02

LLM Fundamentals for Production

Weeks 3–4 · Build Phase

Understand how LLMs actually work at the API level — tokens, context windows, pricing models, and the crucial cost/latency/quality tradeoffs that determine whether your AI product makes or loses money.

🎯 Learning Objectives

  • Compare GPT-4, Claude, Gemini, and Llama for different use cases
  • Implement intelligent model routing based on quality thresholds
  • Build streaming, batching, and caching patterns for LLM APIs
  • Calculate and optimize token costs across providers
  • Design retry-with-backoff strategies for production reliability

📚 Key Topics

  • Tokenization, context windows, and model capabilities
  • Embedding theory and similarity search fundamentals
  • Cost/latency tradeoff analysis across providers
  • API patterns: streaming, batching, caching
  • Model selection frameworks and decision matrices

🛠 Hands-On Project

Build a model router that intelligently selects the cheapest model meeting a quality threshold. Routes requests across GPT-4, Claude, and Llama based on complexity, caches responses, and tracks cost per request in real-time.

OpenAI SDK Anthropic SDK LiteLLM tiktoken
03

Prompt Engineering + Structured Outputs

Weeks 5–6 · Build Phase

Move beyond "write a good prompt" to engineering prompts as versioned, tested, production artifacts. Learn to guarantee structured outputs and build evaluation pipelines that catch prompt regressions before users do.

🎯 Learning Objectives

  • Design system prompts, few-shot chains, and chain-of-thought patterns
  • Enforce structured outputs with Pydantic schemas and JSON modes
  • Build prompt versioning systems with rollback capabilities
  • Create unit evaluations and A/B tests for prompt quality

📚 Key Topics

  • System prompts, persona patterns, and guardrails
  • Pydantic/JSON schema validation for LLM outputs
  • Structured output modes (function calling, tool use)
  • Prompt versioning, A/B testing frameworks
  • Evaluation-driven prompt development

🛠 Hands-On Project

Build a prompt management system with versioning, automated evals, and rollback. Includes a web dashboard showing prompt performance over time, A/B test results, and one-click rollback to previous versions.

Pydantic instructor BAML promptfoo
04

RAG + Knowledge Grounding

Weeks 7–8 · Build Phase

Retrieval-Augmented Generation is the #1 pattern in production AI. Master chunking strategies, hybrid search, reranking, and the metadata engineering that separates toy demos from systems that actually reduce hallucinations.

🎯 Learning Objectives

  • Implement semantic, recursive, and sliding window chunking
  • Build hybrid search combining BM25 + vector similarity
  • Integrate rerankers (Cohere, ColBERT) for precision
  • Design metadata filtering and citation tracking systems
  • Measure and reduce hallucination rates with grounding scores

📚 Key Topics

  • Vector databases: Pinecone, Weaviate, Chroma, pgvector
  • Embedding models and dimension tradeoffs
  • Chunk overlap, parent-child retrieval, contextual compression
  • Metadata schemas for filtering and access control
  • Hallucination detection and confidence scoring

🛠 Hands-On Project

Build a production RAG system over company documentation with inline citations, confidence scores, and source attribution. Includes hybrid search, reranking, metadata filtering, and a hallucination detection pipeline.

LangChain LlamaIndex Chroma sentence-transformers Cohere

AI Workflows + Orchestration

Weeks 9–10 · Ship Phase

Single-shot LLM calls are just the beginning. Learn to build multi-step agent workflows with tool calling, state machines, human-in-the-loop patterns, and the orchestration frameworks powering real AI products.

🎯 Learning Objectives

  • Implement tool calling via function calling and MCP protocols
  • Design state machines for complex multi-step workflows
  • Build human-in-the-loop patterns for high-stakes decisions
  • Create retry logic, fallback chains, and session memory

📚 Key Topics

  • Tool calling patterns and function registration
  • MCP (Model Context Protocol) integration
  • State machines and workflow orchestration
  • Session memory and conversation management
  • Multi-agent coordination patterns

🛠 Hands-On Project

Build a customer support agent that automatically routes tickets, answers common questions from a knowledge base, escalates to humans when confidence is low, and learns from resolved tickets to improve over time.

LangGraph CrewAI Temporal Redis

❓ Frequently Asked Questions

Do I need prior AI/ML experience? ▼

No. The program starts with production Python and builds up systematically. If you can write basic Python (functions, loops, classes), you have enough to start. We send a free 2-week prep course to every enrolled student.

What's the time commitment per week? ▼

Plan for 15–20 hours per week: ~4 hours of live sessions, ~6 hours of project work, and ~5–10 hours of reading, practice, and community participation. Most students balance this with a full-time job.

Is this live or self-paced? ▼

Both. Live sessions run on a fixed schedule (bi-weekly), but all recordings are available immediately. Projects have deadlines but flexible work hours. The community and mentors are available daily.

How is this different from General Assembly, Springboard, etc.? ▼

Three ways: (1) We're 100% focused on Applied AI Engineering — not generic data science or web dev with an AI module bolted on. (2) 70% hands-on — you build 12 production projects, not toy demos. (3) Our curriculum is based on what companies are actually hiring for right now, updated continuously.

What happens if I fall behind? ▼

Life happens. You can pause for up to 2 weeks and rejoin. All recordings are permanent. Your mentor will help you create a catch-up plan. If needed, you can join the next cohort for any stages you missed — free of charge.

Is there a refund policy? ▼

Yes. Full refund within the first 14 days (Stage 1), no questions asked. If you complete all 12 stages, build all projects, and don't land interviews within 6 months, we'll refund 50% of your tuition.
